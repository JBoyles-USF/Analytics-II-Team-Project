\section{Introduction}

Manufacturing defect detection plays an important role in improving product
quality, reducing waste, and preventing costly failures. In manufacturing,
defective components can contribute to equipment damage, increased maintenance,
production delays, customer dissatisfaction, and potential safety risks if they
are not detected early. As manufacturing systems collect more process and test
data, machine learning provides an opportunity to identify defect patterns that
may not be obvious through manual inspection alone.

This project uses historical manufacturing data to predict whether a manufactured part is defective. Of the full dataset, only 0.1991\% of samples are labeled as defective. This makes the problem a severe imbalanced classification task.

The main challenge is that the defective class is extremely rare. This project first compares several machine learning models on this unbalanced data.
Because the defective class is extremely rare, each model is tested with
different imbalance-handling strategies. Different data splits are also compared to
check whether performance remains stable. Finally, models are ranked using a custom FN \& FP Score based on false
negative and false positive rates, since overall accuracy can appear high even
when rare defects are missed. 

The objectives of this project are to:

\begin{enumerate}
    \item Develop a predictive model for classifying defective parts.
    \item Evaluate model performance under severe class imbalance using the custom FN \& FP Score.
    \item Identify key process variables associated with defective production.
    \item Provide insights that can support manufacturing quality control and process improvement.
\end{enumerate}

Together, these objectives outline a framework for using historical manufacturing
data to improve defect detection.

\section{Related Work}
Machine learning has increasingly been applied to manufacturing systems for predictive quality monitoring and defect detection. Prior research shows that industrial process and test data can be used to learn relationships between operating conditions and product quality outcomes.

Zahrani et al.\ apply machine learning models to predict product quality and groove geometry in a circular laser-grooving manufacturing process. Their results show that data-driven models can capture nonlinear relationships between process parameters and final part characteristics, supporting predictive monitoring rather than purely reactive inspection \cite{zahrani2020lasergrooving}.

More recent studies extend this direction using stronger ensemble methods for rare-defect prediction in industrial environments. Geng et al.\ propose a framework for predicting internal cracks in continuous steel casting using CatBoost combined with synthetic data generation, demonstrating improved detection of infrequent defect patterns in a real manufacturing context \cite{geng2025ctgan}.

A central challenge in manufacturing defect prediction is severe class imbalance. Defective products may represent only a small fraction of observations, causing standard training and evaluation to favor the majority class and report high accuracy while missing rare failures. Kafunah et al.\ examine this issue in fault detection and evaluate imbalance-aware strategies such as oversampling and reweighting to improve minority-class detection, highlighting the importance of focusing on defect-class metrics such as precision, recall, and F1 rather than accuracy alone \cite{kafunah2021imbalanced}.

Cost-sensitive learning provides a complementary approach by explicitly assigning higher penalties to misclassifying defective samples, reflecting the higher operational risk of false negatives in manufacturing. Liu et al.\ introduce a cost-sensitive defect detection framework that modifies the training objective using an explicit cost structure to prioritize costly minority-class errors, and they report improved defect identification under imbalance \cite{liu2023costsensitiveyolov5}.

Building on these findings, our project systematically compares interpretable and ensemble models, including logistic regression, decision trees, random forests, and boosting under multiple imbalance-mitigation strategies. We emphasize cost-sensitive random forests and minority-class performance metrics to evaluate model effectiveness for defect detection in a highly imbalanced manufacturing dataset.



\section{Model and Method}

\subsection{Class Imbalance Strategies}
The dataset comprises 229,999 test records with only 458 defective samples (0.2\%). To address the imbalance, we evaluate four dataset manipulation techniques: Random Oversampling (duplicating minority samples), Random Undersampling (removing majority samples), SMOTE (synthetic sample generation), and Class-Weighted training (adjusting learning objectives to penalize minority-class errors). Each data sampling method was tested on every considered model type.
 

\subsection{Considered Model Types}
\subsubsection*{Logistic Regression}
Logistic Regression provides high interpretability, modeling defect probability as a smooth function of sensor readings so engineers can quantify feature impact:
\begin{equation}
P(defective|x) = \sigma(\beta_{0} + \beta^{\top}x) = \frac{1}{1 + e^{-(\beta_{0} + \beta^{\top}x)}}
\end{equation}
\subsubsection*{Decision Trees}
Decision Trees sort data through binary splits based on sensor thresholds. We use Gini Impurity, as it is numerically stable under extreme class imbalance:
\begin{equation}
Gini(S) = \sum_{k \in \{0,1\}} p_{k}(1 - p_{k})
\end{equation}
Splits are evaluated by their reduction in impurity:
\begin{equation}
\Delta = Gini(S) - \left(\frac{|S_{L}|}{|S|}Gini(S_{L}) + \frac{|S_{R}|}{|S|}Gini(S_{R})\right)
\end{equation}
Based on 5-Fold Cross Validation, the optimal maximum depth was set to 5 to prevent overfitting.
 
\subsubsection*{Random Forest}
Random Forest aggregates multiple trees trained on bootstrap samples and random feature subsets,
yielding a majority vote prediction:
\begin{equation}
\hat{y} = \mathrm{mode}\{\hat{y}^{(1)}, \hat{y}^{(2)}, \ldots, \hat{y}^{(N)}\}
\end{equation}
We evaluate both Balanced Random Forest and Cost-Sensitive Random Forest to address the severe class imbalance in the dataset. Balanced Random Forest uses internal undersampling for each tree, meaning each bootstrap sample is adjusted so that the defective class is better represented during training. Cost-Sensitive Random Forest keeps the original training distribution but applies class-weighted impurity, giving higher penalty to mistakes on defective samples. Comparing both approaches allows us to test whether defect detection improves more from changing the sample distribution or from changing the cost of minority-class errors.
 
\subsubsection*{Boosting (AdaBoost, XGBoost, LightGBM, CatBoost)}
Boosting builds an ensemble by repeatedly reweighting samples to focus on previous mistakes. For our baseline, AdaBoost, the error is:
\begin{equation}
\epsilon_i = \sum_{j} w_j^i \delta(C_i(x_j) \neq y_j)
\end{equation}
Classifier importance:
\begin{equation}
\alpha_{t} = \frac{1}{2} \ln\left(\frac{1 - \epsilon_{t}}{\epsilon_{t}}\right)
\end{equation}
Sample weights update:
\begin{equation}
w_{j}^{i+1} = \frac{w_{j}^{i}}{Z_{i}} \times \begin{cases} e^{-\alpha_{i}}, & C_{i}(x_{j}) = y_{j} \\ e^{\alpha_{i}}, & C_{i}(x_{j}) \neq y_{j} \end{cases}
\end{equation}
We extend our analysis to advanced gradient boosting frameworks (XGBoost, LightGBM, and CatBoost) to gauge performance gains and processing efficiency against the AdaBoost baseline.
 
\subsection{Data Split Comparison}

Because the defective class represents only a very small portion of the dataset, model performance may be sensitive to how the data are divided into training and testing sets. A split with a larger training portion, such as 90/10, gives the model more examples to learn from, including more rare defective samples. However, it also leaves fewer defective samples in the test set, which can make metrics more sensitive to small changes in the number of false negatives. In contrast, a 70/30 split provides a larger test set and a more stable estimate of generalization performance, but it reduces the number of defective samples available during training.

To evaluate this trade-off, we compare three stratified train/test split ratios: 90/10, 80/20, and 70/30. Stratification is used so that each split preserves approximately the same defective/non-defective class proportion as the full dataset. This is especially important for this project because random splitting without stratification could produce a test set with too few defective examples, making recall, false negative rate, and the FN \& FP Score unreliable.

The purpose of the split comparison is not only to identify the split with the best score, but also to determine whether model rankings remain stable across different evaluation conditions. If a model performs well only under one split ratio, its performance may be dependent on a favorable partition of rare defective samples. However, if a model remains competitive across multiple splits, this provides stronger evidence that it generalizes to unseen manufacturing data. For this reason, split sensitivity is treated as part of the model evaluation process rather than only a preprocessing decision.

\section{Experimental Design}


\subsection{Model Selection Criteria}
We evaluate simple models alongside complex ensembles to consider the trade-off between interpretability and predictive performance.
Our experimental phase primarily evaluates 120 unique scenarios via our custom "FN \& FP Score" System, as well as secondary evaluation with other relevant metrics in the case that predictive ability (according to our custom scoring system) and interpretability do not significantly emphasize a best choice model.
Our experimental phase evaluates 720 unique scenarios (incorporating 8 models, 5 sampling techniques, 3 train/test splits, and 6 custom weight profiles) via our custom "FN \& FP Score" System. We also conduct secondary evaluations with other relevant metrics (such as PR-AUC and F1-minority score) in the event that predictive ability and interpretability do not significantly emphasize a clear best-choice model.

Conceptual model transparency and interpretability are also heavily considered for final model suggestion/recommendation.
  
\subsection{"FN \& FP Score" System for Evaluating Model Prediction Ability}

In many classification tasks, reducing false negatives (FN) is often more critical than reducing false positives (FP). To systematically evaluate models with this consideration, we propose a weighted scoring system that incorporates both the false negative rate (FNR) and false positive rate (FPR).

\subsubsection{Weighted Score Definition}
We define the \textbf{"FN \& FP Score"} for a model as a weighted sum of the FN Rate and FP Rate:

\begin{equation}
\text{FN \& FP Score} = (\text{FN weighting}) \times (\text{FN Rate}) + (\text{FP weighting}) \times (\text{FP Rate})
\end{equation}

where \(\text{FN weighting}\) and \(\text{FP weighting}\) are non-negative weights reflecting the relative importance of reducing false negatives versus false positives.

\subsubsection{Auxiliary Equation}
Assuming the weights are complementary, we can express the FP weighting in terms of the FN weighting:

\begin{equation}
\text{FP weighting} = 1 - \text{FN weighting}
\end{equation}

\subsubsection{Equivalent Equation}
Substituting the complementary weighting into the score formula yields:

\begin{equation}
\text{FN \& FP Score} = (\text{FN weighting}) \times (\text{FN Rate}) + (1 - \text{FN weighting}) \times (\text{FP Rate})
\end{equation}

This equivalent equation shows that the score is a function of three components: FN weighting, FNR, and FPR.


\subsubsection{Model Ranking Using the "FN \& FP Score"}
The "FN \& FP" Score allows ranking models according to their performance while prioritizing reduction of false negatives. For a given FN weighting, the \textbf{lowest score} indicates the \textbf{best model} at that weighting.
By varying FN weighting (e.g., 50\%, 60\%, …, 100\%), we can evaluate model performance under different prioritization schemes.


This approach provides a flexible and interpretable framework for model selection, emphasizing the trade-off between FN and FP according to domain-specific or safetyrequirements.
 
\subsubsection{Subjectivity of FN Weighting}

The FN weighting metric is subject to change based on organizational priorities. For example, incorporating the severe ethical or safety consequences of missing a defect could heavily increase the FN weighting. A completely unbiased, cost-based method for calculating the FN weighting is as follows:

\begin{equation}
\text{FN weighting} = \frac{\text{Cost of FN}}{\text{Cost of FN} + \text{Cost of FP}}
\end{equation}

In this case, the respective costs could represent labor, material waste, system downtime, or reputational damage to the company. 

\subsection{Other Relevant Metrics}
Due to the extreme class imbalance (99.8\% non-defective to 0.2\% defective), standard accuracy and ROC AUC can be highly misleading \cite{imani2026roc}. Instead, recall, precision, Area Under the Precision-Recall Curve (PR-AUC), and the F1-minority score are used as secondary metrics to evaluate our models.
Summary visualizations of these metrics are embedded within our attached Jupyter Notebook. These metrics serve as a vital secondary evaluative criteria to ensure the models aren't achieving high recall simply by blindly flagging every item as a defect.

\section{Results}

\subsection{Comparing Predictive Ability}
Bar charts displaying the top 10 scoring models per FN weighting are provided in Appendix B. In this context, a lower score indicates superior predictive ability for defect detection at a specific weight. 

The results of our study demonstrate that a class-weighted logistic regression model with an 80/20 split outperforms all other tested models at 50\% and 60\% FN weightings, and ranks second best across all other weightings. For the 70\%, 80\%, 90\%, and 100\% FN weightings, a random undersampling AdaBoost model with a 90/10 split achieves the best predictive performance.

\subsection{Comparing Interpretability}

All models other than logistic regression that ranked within the top 10 for each weighting category utilize some form of boosting. While these ensemble techniques demonstrate strong predictive performance, the logistic regression model provides superior interpretability compared to its boosting-based counterparts. This high level of transparency is particularly advantageous for defect detection, as it allows stakeholders to easily interpret the influence of individual features on the model's output.

This clarity comes from the fact that logistic regression represents predictions as a weighted combination of input features, where each factor has a direct and interpretable contribution. In contrast, boosting models combine many decision rules across multiple trees, creating more complex interactions that are harder to interpret at a glance. While these models are often more powerful, their structure can obscure the overall reasoning process. By remaining relatively simple, logistic regression allows experts to more easily understand which factors influence defects, without relying heavily on additional interpretation tools.

\subsection{Secondary Metric Comparison}

Secondary metrics were reviewed to confirm whether the top-ranked models under the FN \& FP Score also performed well under standard imbalanced-classification measures. Recall, precision, F1-minority, PR-AUC, and ROC-AUC comparisons are shown in Appendix B, Figures \ref{fig:recall_precision_tradeoff}--\ref{fig:peak_recall_model}.

As shown in Figure \ref{fig:peak_recall_model}, AdaBoost with random undersampling achieved the highest peak recall at 0.9545. Class-weighted Logistic Regression followed closely at 0.9432, while LightGBM, XGBoost, and CatBoost each reached 0.9318 under their best configurations. This means the recommended Logistic Regression model gave up only about 0.0113 recall compared with the strongest recall model, while maintaining much stronger interpretability. The recall heatmap in Figure \ref{fig:recall_heatmap} also shows that random undersampling produced the strongest recall for several ensemble models, including AdaBoost, CatBoost, XGBoost, and LightGBM. However, Logistic Regression performed best with class weighting, suggesting that adjusting the learning penalty was more effective for the linear model than changing the sample distribution.

The recall-versus-precision plot in Figure \ref{fig:recall_precision_tradeoff} shows why recall alone was not enough to select the final model. Several high-recall configurations had noticeably lower precision, meaning they detected many defective samples but also produced more false alarms. The strongest 80/20 class-weighted Logistic Regression configuration achieved recall of 0.9432, precision of 0.0647, F1-minority of 0.1212, PR-AUC of 0.8019, and ROC-AUC of 0.9880. A similar 90/10 class-weighted Logistic Regression configuration had slightly lower recall at 0.9318, but higher precision at 0.0858 and higher F1-minority at 0.1571. Overall, the secondary metrics support the final recommendation because Logistic Regression remained competitive across standard minority-class metrics while remaining easier to interpret than the strongest boosting models.
\section{Conclusion and Future Work}

\subsection{Our Recommended Model}

Our recommended model across all FN weights is Class-weighted Logistic Regression with a 80/20 split.

\begin{itemize}
    \item \textbf{Performance:} Consistently achieves top-tier results, ranking first for $50\%$ and $60\%$ weights and at least second across all other categories.
    \item \textbf{Interpretability:} Provides greater transparency and feature insight than AdaBoost, which is the only model to exceed its performance at weights $\ge 70\%$.
\end{itemize}

\subsection{Limitations of Our Study} 
This study has several methodological and practical limitations. We benchmarked our models across 720 scenarios to ensure robust validation. However, we encountered a structural limitation regarding the distinction between \textit{cost-sensitive learning} and \textit{cost-sensitive evaluation}. Because our custom weights were applied via \texttt{make\_scorer} in \texttt{GridSearchCV}, the formula acted purely as an external evaluator rather than being passed directly into the \texttt{.fit()} function. Consequently, the models used a default 50\% decision threshold on balanced data, resulting in hypersensitive predictions that flagged a large portion of the dataset to avoid false negatives. Despite this, our custom formula successfully standardized the selection process post-prediction, providing a robust platform to equally compare how well the algorithms handled imbalance while maintaining interpretability. Beyond the modeling architecture, the dataset itself presents challenges. It is extremely imbalanced (only 458 defects out of nearly 230,000 records) and consists of numerical features lacking contextual descriptions, limiting generalization and requiring domain knowledge for practical interpretation. Finally, the static nature of the dataset does not account for real-time production variability. 
\subsection{Potential Future Improvements} 
Future work should transition from post-training evaluation to true cost-sensitive optimization. Instead of retraining models for every weighting scenario, the pipeline should either dynamically inject weights directly into the algorithms during training (e.g., via \texttt{class\_weight}), or utilize post-training threshold tuning (sliding the defect trigger below 50\%) to optimize the decision boundary without redundant computation. Additionally, incorporating more defective sample data across different operating conditions will help the models better learn minority-class patterns. Finally, deploying the selected models in a real-time, streaming environment is necessary to evaluate their performance against production drift and determine periodic retraining intervals.
